## Description

Provide a structured pipeline that turns citizen-identified problems into competing, fully-developed proposals. Any citizen can register an issue; issues that reach a population-scaled support threshold enter a development phase where multiple competing solutions can coexist.

## Acceptance Criteria

- [ ] Any citizen can submit problems, opportunities, and improvement suggestions
- [ ] Issues are publicly visible and support evidence, comments, and endorsements
- [ ] Proposals must pass a population-scaled support threshold before reaching a vote
- [ ] Multiple competing proposals can address the same problem

Source: White Paper System Design Layers 3-4 (Problem Registry, Proposal Marketplace) and Module 4 proposal submission process. Implements FR-015 through FR-019.
