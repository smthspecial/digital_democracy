## Responsibility

Manages budget categories, citizen-directed allocation votes, and the real-time public ledger of inflows and outflows. Provides the funding information gate for proposals (together with proposal_budget owned by the proposal-service). Supports participatory budgeting (FR-038) and maintains a continuously public, real-time view of government finances (FR-036).

## Owned Tables

| Table | Role |
|-------|------|
| TBL-026 budget_category | Primary owner — hierarchical category tree per jurisdiction |
| TBL-027 budget_allocation_vote | Primary owner — citizen percentage preferences per category per period |
| TBL-028 ledger_entry | Primary owner — immutable inflow/outflow records |

## Operations

| Operation | Mode | Process |
|-----------|------|---------|
| Submit budget allocation vote | sync | DP-013 |
| Record ledger entry | sync | DP-019 |
| Budget allocation aggregation | cron | DP-051 |
| Ledger reconciliation | cron | DP-055 |

## Key rules

- `budget_category` is hierarchical (self-referential `parent_id`). Citizens allocate within categories (FR-039), not a flat pool; this reduces zero-sum framing.
- `budget_allocation_vote` is unique per `(citizen_id, category_id, period)`. Totals per citizen per period must sum to 100%; enforced at write time.
- DP-051 aggregates all `budget_allocation_vote` rows for the closing period, computes weighted-average allocations, and writes the result back to `budget_category.allocated_amount`. The aggregation result is public.
- `ledger_entry` is append-only. Operators write inflows and outflows (DP-019) and they are immediately public (FR-036). No update or delete permitted; corrections are new compensating entries.
- DP-055 runs daily to reconcile total outflows against `budget_category.allocated_amount` and surfaces discrepancies as alerts for the audit pool.

## Dependencies

- Emits to: `audit-service` (DP-036), `notification-service` (DP-039 for allocation period open/close)
- Read by: `proposal-service` (funding gate before voting), `project-service` (budget tracking), `civic-duty-service` (audit assignments via DP-054)
