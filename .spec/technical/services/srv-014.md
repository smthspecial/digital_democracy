## Responsibility

Maintains a per-citizen reputation signal that reflects quality of governance participation: accurate predictions, constructive arguments, transparent disclosures, and successful proposals yield positive delta; misinformation, undisclosed conflicts, manipulation, and fraud yield negative delta. Reputation is entirely separate from competency and never alters voting weight (FR-027).

## Owned Tables

| Table | Role |
|-------|------|
| TBL-016 reputation_record | Primary owner — signed delta events per citizen per factor type |

## Operations

| Operation | Mode | Process |
|-----------|------|---------|
| Reputation score update | async | DP-038 (triggered by outcome evaluated, challenge resolved, contribution logged) |

## Key rules

- `reputation_record` is an event log (one row per factor event), not a single mutable score. Current reputation is the sum of all `delta` values for a citizen. This preserves the full history.
- `factor_type` enum covers both positive (`accurate_prediction`, `constructive`, `disclosure`, `successful_proposal`) and negative (`misinformation`, `undisclosed_conflict`, `manipulation`, `fraud`) factors (FR-027).
- Reputation never affects voting weight, competency status, or assignment priority (FR-027, FR-052). It is a public informational signal only.
- DP-038 is triggered by multiple upstream services: voting-service (participation signal), project-service (outcome success/failure), competency-service (challenge upheld/dismissed), deliberation-service (constructive contribution signal).
- Writing a negative reputation delta requires an upstream authoritative decision (e.g., a competency challenge upheld via DP-032, or a fraud finding from an audit). Self-reported negative deltas are not permitted.

## Dependencies

- Receives events from: `voting-service`, `project-service`, `competency-service`, `deliberation-service`
- Emits to: `audit-service` (DP-036), `notification-service` (DP-039 on significant delta)
- Read by: public citizens (informational display), `competency-service` (context for challenge review)
