## Responsibility

Tracks the implementation of approved proposals as public projects. Provides real-time visibility into timelines, budget spend, contractors, and milestone completion. Triggers outcome evaluations after project completion. Enforces the principle that citizens decide what is done while professional institutions decide how it is done (FR-045).

## Owned Tables

| Table | Role |
|-------|------|
| TBL-029 project | Primary owner — proposal-to-project mapping, contractor, budget, status |
| TBL-030 project_milestone | Primary owner — ordered milestones with due dates and completion |
| TBL-031 outcome_evaluation | Primary owner — promised vs measured outcomes |

## Operations

| Operation | Mode | Process |
|-----------|------|---------|
| Report project milestone | sync | DP-018 |
| Submit outcome evaluation | sync | DP-022 |
| Outcome evaluation trigger | cron | DP-053 |

## Key rules

- A `project` row is created when a `proposal` transitions to `approved`. It inherits `budget_allocated` from `proposal_budget.cost` and links back via `proposal_id`.
- `budget_spent` is updated incrementally as `ledger_entry` outflows (owned by budget-service) are reconciled against the project (DP-055 in budget-service).
- `project_milestone` rows are created when the project is instantiated and updated as the operator reports completion (DP-018). All updates are public and emitted to the audit log.
- DP-053 runs daily. When a project reaches `status=completed` and the configured evaluation delay has passed (e.g., 6 months post-completion), it enqueues DP-040 to generate an outcome evaluation assignment for the audit pool.
- `outcome_evaluation` compares `promised_outcome` (written at proposal approval) to `measured_outcome` (written by the audit/oversight team at evaluation time). Result is public (FR-048).
- `contractor` is a public field (FR-047). No private contracting relationships.

## Dependencies

- Emits to: `audit-service` (DP-036), `notification-service` (DP-039: milestone delays, completion), `civic-duty-service` (DP-053 triggers evaluation assignments)
- Reads from: `proposal-service` (approved proposal data), `budget-service` (ledger reconciliation)
