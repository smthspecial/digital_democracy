## Responsibility

Maintains the append-only hash-chained audit log, manages constitutional rights definitions, and executes constitutional review of proposals before they enter voting. Is the system's integrity backbone: every service writes to this service; no service writes back to it. Supports redundant independent auditing by being readable by multiple concurrent audit bodies (FR-066).

## Owned Tables

| Table | Role |
|-------|------|
| TBL-034 audit_log | Primary owner — append-only, hash-chained, publicly verifiable event record |
| TBL-035 constitutional_right | Primary owner — protected rights outside ordinary voting |
| TBL-036 constitutional_review | Primary owner — per-proposal per-right review results |

## Operations

| Operation | Mode | Process |
|-----------|------|---------|
| Audit log append | async | DP-036 (consumed from audit.append queue) |
| Constitutional review trigger | async | DP-034 (triggered when proposal enters voting) |
| Protocol change delayed execution gate | async | DP-043 |

## Key rules

- `audit_log` is append-only. No service may update or delete rows. Tampering is detectable via `prev_hash` chain integrity (ADR-005, FR-060).
- Each `audit_log` row is hash-chained: `prev_hash` = hash of the previous row; `payload_hash` = hash of the action payload; `signature` = service key signature. Gap detection is automated.
- Ballot content is never written to the audit log (NFR-001); only session-level events (session opened, tally certified) are recorded.
- The `audit.append` queue has at-least-once delivery with deduplication. Ordering within a sequence is enforced by the service using the hash chain; out-of-order arrivals are held until the predecessor is written.
- Constitutional review (DP-034): for every proposal entering voting, this service compares the proposal against all `constitutional_right` rows where `protected=true`. If any right is affected, it writes a `constitutional_review` row with `result=blocked` and rejects the vote session creation. Cleared proposals get `result=cleared`. Blocked proposals route to an elevated review process.
- DP-043 gates protocol changes: the service verifies all required approvals exist, the delay period has elapsed, and the change has been publicly visible; then releases the execution signal.
- Independent audit bodies read this service's tables directly; they cannot write. Multiple concurrent readers are explicitly supported (FR-066).

## Dependencies

- Receives from: every other service via `audit.append` queue and direct calls for constitutional review
- Emits to: `notification-service` (on blocked constitutional review, DP-039), `governance-role-service` (protocol change activation, DP-043)
