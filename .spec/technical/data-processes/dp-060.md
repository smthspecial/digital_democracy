## Description

Enrolls a new MFA factor (TOTP, passkey, or facial biometric) for a citizen.

### TOTP enrollment

1. auth-service generates a TOTP secret, returns QR code to client (one-time display only).
2. Citizen scans with authenticator app and submits the first code to confirm.
3. On valid code: encrypt secret, write `mfa_factor` row (TBL-038, factor_type=totp, status=active).
4. Secret is stored encrypted; plaintext is discarded after enrollment confirmation.

### Passkey enrollment

1. auth-service generates a WebAuthn credential creation challenge.
2. Client performs platform or hardware authenticator ceremony; returns credential ID and public key.
3. auth-service verifies the attestation.
4. Write `mfa_factor` row (TBL-038, factor_type=passkey, status=active).
5. Private key never leaves the citizen's device.

### Facial biometric enrollment

1. Client captures a live facial video (liveness detection frame sequence).
2. auth-service extracts a biometric embedding from the capture.
3. Cross-references against the reference embedding captured during identity verification (DP-002, stored in TBL-038 from that initial capture).
4. If the embeddings match above the confidence threshold: write `mfa_factor` row (TBL-038, factor_type=facial, status=active, biometric_embedding_enc = new embedding).
5. Raw video frames are discarded immediately after embedding extraction.
6. If confidence is below threshold → enrollment rejected; citizen prompted to retry or use a different factor type.

### On any enrollment success

- Write auth_event (TBL-039, event_type=factor_enrolled).
- Emit to audit.append (DP-036).
- Upgrade current session to T2 immediately (or T3 if facial or passkey was enrolled).
- Emit notification to citizen confirming new factor (DP-039).
