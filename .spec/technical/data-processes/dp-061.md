## Description

Issues and validates an MFA challenge to raise the session's assurance tier. Called at login (by DP-059) and by any service that detects the current access token's `assurance_tier` is below the required tier for the requested operation (AUTH-010).

### Challenge flow

1. auth-service determines which factors the citizen has enrolled (TBL-038, status=active).
2. For T2: any enrolled factor is acceptable. Present available options to the client.
3. For T3: facial biometric or passkey is required. If neither is enrolled → challenge fails with enrollment prompt.
4. Client completes the challenge:
   - TOTP: submits 6-digit code; auth-service validates against encrypted secret (with drift tolerance ±1 window).
   - Passkey: completes WebAuthn assertion ceremony; auth-service verifies signature against stored public key.
   - Facial: client submits live capture; auth-service runs liveness detection then embedding comparison against TBL-038.biometric_embedding_enc. Match must exceed confidence threshold.
5. On success:
   - Update `session.assurance_tier` and `session.last_mfa_at` (TBL-037).
   - Issue a new short-lived access token with updated tier.
   - Write auth_event (TBL-039, event_type=stepup_success).
   - Emit to audit.append (DP-036).
6. On failure:
   - Write auth_event (TBL-039, event_type=stepup_failure or mfa_failure).
   - After 5 consecutive failures within 10 minutes: trigger DP-066 (anomaly, reason=mfa_brute_force), suspend session.
   - Emit to audit.append (DP-036).

### Tier validity windows (ADR-014)

| Tier | MFA validity window | Re-challenge trigger |
|------|--------------------|--------------------|
| T2 | 12 hours from last MFA | `last_mfa_at` > 12h ago |
| T3 | 5 minutes from last MFA | `last_mfa_at` > 5min ago |

T3 window means: for casting a ballot or submitting a protocol approval, the citizen must have completed a biometric or passkey challenge within the last 5 minutes. The service requesting the action checks the token's `assurance_tier` and `last_mfa_at`; if stale, it returns a 403 with a step-up required signal and the client triggers DP-061.
