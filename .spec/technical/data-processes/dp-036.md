## Description

Appends an entry to the hash-chained `audit_log`. Each entry carries `prev_hash`, `payload_hash`, and a service signature. Delivery: at-least-once with deduplication. Ordering is enforced by the audit-service using the hash chain; out-of-order arrivals are held until the predecessor is committed. Ballot content is never logged.
