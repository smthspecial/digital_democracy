## Columns

| Column | Type | Description |
|--------|------|-------------|
| id | uuid | Primary key |
| citizen_id | uuid | Factor owner |
| factor_type | enum | totp \| passkey \| facial |
| status | enum | active \| revoked |
| totp_secret_enc | text | Encrypted TOTP secret (null for non-TOTP factors) |
| passkey_credential_id | text | WebAuthn credential ID (null for non-passkey factors) |
| passkey_public_key | text | WebAuthn public key (null for non-passkey factors) |
| biometric_embedding_enc | text | Encrypted facial embedding (null for non-facial factors); raw image never stored |
| enrolled_at | timestamptz | Enrollment time |
| last_used_at | timestamptz | Last successful verification time |
| revoked_at | timestamptz | Revocation time (null if active) |

## Notes

Implements ADR-014. A citizen may hold multiple factors of different types. `biometric_embedding_enc` stores an encrypted embedding derived from the reference capture in DP-002; the raw image is discarded after extraction. The embedding never leaves auth-service. Revoked factors are retained for audit.
