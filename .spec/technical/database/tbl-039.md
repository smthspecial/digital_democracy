## Columns

| Column | Type | Description |
|--------|------|-------------|
| id | uuid | Primary key |
| citizen_id | uuid | Actor (null for unauthenticated login attempts) |
| session_id | uuid | Associated session (null before session creation) |
| event_type | enum | login_success \| login_failure \| mfa_success \| mfa_failure \| stepup_success \| stepup_failure \| anomaly_detected \| session_revoked \| factor_enrolled \| factor_revoked |
| factor_type | enum | totp \| passkey \| facial \| null |
| ip_address | text | Originating IP (stored for anomaly investigation; not public) |
| device_fingerprint | text | Device fingerprint at event time |
| anomaly_reason | text | Populated on anomaly_detected events (new_device \| new_country \| concurrent_geos \| mfa_brute_force \| token_reuse) |
| created_at | timestamptz | Event time |

## Notes

Implements ADR-014. All auth_event rows are emitted to audit.append (DP-036) as identity_event entries. ip_address and device_fingerprint are retained for anomaly investigation but are never exposed publicly. Failed login attempts are recorded to support brute-force detection. Rows are immutable after creation.
