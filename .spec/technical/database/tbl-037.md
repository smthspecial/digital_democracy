## Columns

| Column | Type | Description |
|--------|------|-------------|
| id | uuid | Primary key |
| citizen_id | uuid | Session owner |
| access_token_hash | text | Hash of the current short-lived access token (15-min TTL) |
| refresh_token_hash | text | Hash of the rotating refresh token (server-side only) |
| device_fingerprint | text | Fingerprint of the device that initiated the session |
| ip_subnet | text | /24 subnet of the originating IP at session creation |
| assurance_tier | enum | T1 \| T2 \| T3 — current MFA assurance level |
| last_mfa_at | timestamptz | Time of last successful MFA verification |
| last_refresh_at | timestamptz | Time of last token refresh |
| expires_at | timestamptz | Refresh token expiry (absolute session deadline) |
| status | enum | active \| suspended \| revoked |
| created_at | timestamptz | Session creation time |

## Notes

Implements ADR-014. Refresh tokens are rotated on each use; the previous hash is invalidated immediately. A device_fingerprint or ip_subnet mismatch on refresh triggers DP-066 (anomaly). `status=revoked` rows are retained for audit; they cannot be reactivated.
